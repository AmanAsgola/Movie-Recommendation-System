How It Works

Three engines.
One perfect answer.

No ML background required. Here's the intuition behind each algorithm.

🎯

Content-Based
Filtering

Analyzes what movies are — their genres, themes, and metadata — converting them into TF-IDF vectors. Finds films that are mathematically closest to what you already love using cosine similarity.

"You liked Inception? Here are other mind-bending thrillers with similar genre DNA — no user data needed."
TF-IDF Vectors K-Nearest Neighbors Cosine Similarity Fuzzy Title Match
👥

Collaborative
Filtering (SVD)

Studies who rated what. Singular Value Decomposition discovers hidden patterns across 20M+ ratings — finding users with identical taste profiles and surfacing unseen gems.

"Users just like you — who also loved Inception — all rated Interstellar and The Prestige in their top 5."
SVD Matrix Factorization 30 Latent Factors scikit-surprise 20M Ratings
⚡

Hybrid
Engine

Runs both algorithms in parallel and applies a 1.2× boost multiplier to movies appearing in both outputs. The result is recommendations that are both deeply personal and contextually relevant.

"Your taste-matched friend who also knows your genre preferences — giving you a joint, calibrated top 10."
Score Fusion 1.2× Boost Top-50 Pool Cold-Start Safe

Product Lifecycle

From prototype
to production.

Phase 1 · Data Foundation

Loaded and cleaned the MovieLens 20M dataset — 27,278 movies, 138,493 users. Built genre + title feature vectors using TF-IDF. Established the content similarity matrix using sparse cosine distance.

Phase 2 · Model Development

Trained SVD collaborative filtering on 30K sampled ratings. Tuned 30 latent factors and 5 epochs to balance accuracy vs. compute. Implemented the 1.2× boost fusion logic for the hybrid engine.

Phase 3 · Application Layer

Built the Streamlit web interface with real-time TMDB poster fetching. Integrated RapidFuzz (75% threshold) for forgiving title search. Deployed a 5-column responsive results grid.

Phase 4 · Evaluation

Implemented Precision@10 evaluation framework. Achieved 42% precision on held-out test users. Documented that the hybrid approach consistently outperforms either standalone model.

Why We Built This

Built out of
frustration.

Every recommendation tutorial online stops at a Jupyter notebook. They show you the math, then leave you with no UI, no real data, no evaluation, no fuzzy search, no posters — nothing deployable.

We built this system end-to-end because we wanted something you could actually show to a user — not just a matrix of similarity scores. That means a real web app, real movie posters, real fuzzy search, and real evaluation numbers.

The gap in the market isn't just in production code — it's in transparent hybrid systems that any developer or data scientist can fork, understand, and adapt. This is our answer to that gap.
